Implement PetStore application Controllers

In this lesson step we will add controller logic to the Controller projects in the PetStore solution. In most applications we have a persistence layer, often called the repository layer. In this tutorial we create a PetStoreRepo project to use DynamoDB to store our data and then configure our Controller classes to use that PetStoreRepo. Since our PetStoreRepo will use AWS DynamoDB, we provide a project template called LazyStackDynamoDBRepo that implements generic Create, Read, Update, Delete and List (CRUDL) operations on the Amazon DynamoDB. We will use this LazyStackDynamoDBRepo in our PetStoreRepo project.

Create the DynamoDBRepo project

  1. Right Click on the "Solution PetStore" item in the solution explorer. The context menu appears.
  2. Select Add → New Project. The "Add a new project" dialog appears.
  3. Type "LazyStackDynamoDBRepo" into the search field. A list of project templates is presented.
  4. Double Click on the LazyStackDynamoDBRepo project template. The "Configure your new project" dialog appears.
  5. Enter "DynamoDBRepo" as the Project Name.
  6. Click on the Create button. A solution with your DynamoDBRepo project is created.
  7. Select Visual Studio menuitem Build → Rebuild Solution. The solution compiles successfully.

The DynamoDB project implements a very generalized and highly-reusable strategy for managing tables in DynamoDB. See the LazyStack DynamoDB documentation for more details.

Create the PetStoreRepo project

  1. Create PetStoreRepo project from "C# Class Library (.NET Core)" template.
    1. Right Click on Solution PetStore item in the solution explorer. The context menu appears.
    2. Select the Add → New Project context menu item. The "Add a new project" dialog appears.
    3. Find the "C# Class Library (.NET Core)" project template.
    4. Double click on the "C# Class Library (.NET Core)" template. The Configure your new project window is presented.
    5. Enter PetStoreRepo" as the project name.
    6. Click the Create button. The project is created with a default class Class1.cs.
  2. Delete the Class1.cs file from the PetStoreRepo project.
  3. Right Click on the PetStoreRepo project. The context menu appears.
  4. Select Add → New Folder. A "New Folder" is created in the project.
  5. Enter "Models" as the folder name.
  6. Click on the following class files to download them and then move them into the Models folder.
    • CategoryRepo.cs
    • OrderRepo.cs
    • PetRepo.cs
    • TagRepo.cs
  7. Add Project References
    1. Right click on Dependencies. The context menu appears.
    2. Select Add Project Reference. The Reference Manager dialog appears.
    3. Select these projects:
      • OrderController
      • OrderSecureController
      • PetController
      • PetSecureController
      • DynamoDBRepo
    4. Click the OK button.
  8. Build the project. The project builds successfully.

Step Summary

In this step we implemented controller logic by:

  • Creating a DynamoDBRepo project using the LazyStackDynamoDB project template.
  • Creating a PetStoreRepo project using the C# Class(.NET Core) Library template.
  • Implementing the PetStoreRepo repository logic with the CategoryRepo.cs, OrderRepo.cs, PetRepo.cs and TagRepo.cs files.
  • Configuring the PetStoreRepo projects references.
