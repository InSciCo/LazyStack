Schema Project

LazyStack uses the OpenApiGenerator tool to generate a schema project (PetStoreSchema in our example). This project contains class definitions supporting the OpenApi schema. This project is fully regenerated each time you run LazyStack so you never edit this project's source code.

The schema project is a dependency of both the client and service-side projects. Using the same library ensures both sides of your application have a consistent view of the data exchanged between them. It is not unusual for client and service-side projects to have additional data classes but these may not be included in the schema project.
