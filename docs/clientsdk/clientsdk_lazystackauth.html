LazyStackAuth Library

LazyStackAuth is a library that you can use from a C# app to authenticate with a LazyStack application stack. There is no special or proprietary sauce here but the library does make some assumptions about the configuration of your AWS Cognito resources as specified in the serverless.template file generated by LazyStack. See the tutorial for examples of using LazyStackAuth and ClientSDK in a .NET Console app and Xamarin App.

Single Page Xamarin Authentication Model

LazyStackAuth contains the AuthProcess class. AuthProcess exposes bindable properties and events that make it easy to authentication using a C# UI. A fully functional example is provided in the tutorial Xamarin App. The app uses a single Xamarin Page to handle all authentication interactions making it easy to replicate in your own apps.

State Driven Authentication

LazyStackAuth is a state driven authentication machine implementing a iterative authentication challenge process. The properties and methods of the AuthProcess class are largely self-explantory so we introduce the AuthProcess class by presenting the simplest happy-path use-cases;

Sign In Happy Path using AuthProcess class
  1. SignedIn == false // must be signed out to start sign in
  2. StartSignInAsync();
  3. // Collect login from user and assign it to Login.
  4. VerifyLoginAsync();
  5. // Collect password from user and assign it to Password.
  6. VerifyPassword();
  7. SignedIn == true
Sign Up Happy Path using AuthProcess class
  1. SignedIn == false // must be signed out to start sign up
  2. StartSignUpAsync()
  3. // Collect login from user and assign it to Login.
  4. VerifyLoginAsync();
  5. // Collect password from user and assign it to Password.
  6. VerifyPasswordAsync(); // AWS Sends validation code to user's email.
  7. // Collect validation code and assign it to Code.
  8. VerifyCodeAsync();
  9. // The user can now Sign In.
Reset Password Happy Path using AuthProcess class
  1. IsSignedIn == false // must be signed out to do password reset
  2. StartResetPasswordAsync()
  3. // Collect login from user and assign it to Login.
  4. VerifyLoginAsync();
  5. // Collect new password from user and assign it to NewPassword.
  6. VerifyNewPasswordAsync(); // AWS Sends validation code to user's email.
  7. // Collect validation code and assign it to Code.
  8. VerifyCodeAsync();
  9. // The user can now Sign In with the new password.
Update Password Happy Path using AuthProcess class
  1. IsSignedIn == true // must be signed in to update password
  2. StartUpdatePasswordAsync();
  3. // Collect old password and assign it to Password.
  4. VerifyPasswordAsync();
  5. // Collect new password and assign it to NewPassword.
  6. VerifyNewPasswordAsync();
  7. // Password has been changed and should be used on next Sign In.
Update Email Happy Path using AuthProcess class
  1. IsSignedIn == true // must be signed in to update email
  2. StartEmailUpdateAsync();
  3. // Collect new email and assign it to Email.
  4. VerifyEmailAsync(); // AWS sends validation code to user's new email.
  5. // Collect validation code and assign it to Code.
  6. VerifyCodeAsync();
  7. // Email has been updated.

Comprehensive State Management and Events

The AuthProcess class implements INotifyPropertyChanged so properties are bindable and raise property changed events as actions are taken in the class. In addition, the class emits events for each change of state. The AuthProcess class implements the IAuthProvider interface. The AuthProcess class is a wrapper around any class that implements the IAuthProvider interface. We provide the AuthProviderCognito implementation of the IAuthProvider interface.
